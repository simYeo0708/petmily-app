다음의 작업들을 수행해줘. 상품 추천은 인기 상품 top5 바로 위에 사용자가 스와이프 이벤트를 통해 상품들을 넘겨볼 수 있게 ui를 구현하기
상품 추천 기능 구현은 LLM - Back - DB 프로세스로 이루어질 것이고, 2번의 ui와 연결시키기
구현된 채팅기능 테스트해보기 세션 하나를 새로파서, 서로 다른 주소(서로다른 사용자간의) 소통이 가능한지 테스트하기. 상품 추천의 경우는 다음의 내용처럼 프론트와 백 연결해서 구현하기. LLM이 직접 DB에 접속하는 게 아니라,
LLM ⟷ (우리 백엔드 API) ⟷ DB
이 구조로 가야 해.
핵심 개념: “툴(함수) 호출” / “RAG + 함수 호출”
흐름은 이렇게 만들면 돼:
사용자가 앱에서 “몽이 추천 상품 보여줘” 요청
백엔드에서:
pet_id로 Pet Profile 가져옴
상품 후보군을 DB에서 쿼리해서 가져옴
이 정보들을 LLM에게 넘긴다 (프롬프트 + JSON)
아까 내가 만들어준 system prompt + user prompt 템플릿 활용
LLM이:
받은 상품 리스트 안에서만 고르고
추천 이유까지 달아서 JSON으로 응답
백엔드가 이 JSON을 파싱해서 프론트에 전달 → 앱에서 보여줌
여기서 “DB 접근”은 항상 너희 서버가 담당하고,
LLM은 “주어진 데이터 안에서 판단만 하는 역할.”
