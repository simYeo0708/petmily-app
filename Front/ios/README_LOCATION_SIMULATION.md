# iOS 시뮬레이터 위치 시뮬레이션 가이드

## 🎯 빠른 시작

### 방법 1: Xcode 시뮬레이터 메뉴 (가장 간단)

1. **시뮬레이터 실행 후 상단 메뉴 클릭:**
   ```
   Features > Location > Custom Location...
   ```

2. **좌표 입력:**
   - Latitude: `37.5665` (서울시청)
   - Longitude: `126.9780`

3. **자동 이동 시뮬레이션:**
   - `Features > Location > Freeway Drive` - 고속도로 주행
   - `Features > Location > City Run` - 도시 달리기
   - `Features > Location > City Bicycle Ride` - 자전거 주행

### 방법 2: GPX 파일 사용 (실제 경로 시뮬레이션)

1. **GPX 파일을 Xcode 프로젝트에 추가:**
   - Xcode에서 `ios/Petmily.xcworkspace` 열기
   - `seoul-walk-simulation.gpx` 파일을 프로젝트에 드래그 앤 드롭
   - "Copy items if needed" 체크

2. **스킴 설정:**
   - Xcode 상단에서 스킴 선택
   - `Edit Scheme...` 클릭
   - `Run` > `Options` 탭
   - `Default Location`에서 `seoul-walk-simulation.gpx` 선택

3. **앱 실행:**
   - 앱을 실행하면 GPX 파일의 경로를 따라 자동으로 위치가 업데이트됩니다

## 🔧 고급 설정

### 위치 업데이트 속도 조절

GPX 파일에서 `<time>` 태그 간격을 조절하여 이동 속도를 시뮬레이션할 수 있습니다:

```xml
<!-- 빠른 이동 (10초 간격) -->
<trkpt lat="37.4980" lon="127.0276">
  <time>2024-01-01T10:00:00Z</time>
</trkpt>
<trkpt lat="37.4990" lon="127.0286">
  <time>2024-01-01T10:00:10Z</time>
</trkpt>

<!-- 느린 이동 (1분 간격) -->
<trkpt lat="37.4980" lon="127.0276">
  <time>2024-01-01T10:00:00Z</time>
</trkpt>
<trkpt lat="37.4990" lon="127.0286">
  <time>2024-01-01T10:01:00Z</time>
</trkpt>
```

### 여러 경로 테스트

다양한 GPX 파일을 만들어 테스트할 수 있습니다:
- `short-walk.gpx` - 짧은 산책 (5분)
- `long-walk.gpx` - 긴 산책 (30분)
- `circular-walk.gpx` - 원형 경로

## ⚠️ 주의사항

1. **시뮬레이터 위치 설정은 앱 재시작 시 초기화됩니다**
   - 앱을 다시 실행하면 마지막 설정한 위치로 돌아갑니다

2. **GPX 파일은 Xcode 프로젝트에 포함되어야 합니다**
   - 파일이 프로젝트에 없으면 스킴 설정에서 선택할 수 없습니다

3. **실제 GPS 정확도와 다를 수 있습니다**
   - 시뮬레이터는 완벽한 정확도로 위치를 제공합니다
   - 실제 기기에서는 GPS 노이즈와 정확도 문제가 발생할 수 있습니다

## 🚀 테스트 시나리오

### 시나리오 1: 기본 위치 추적
1. `Custom Location...`으로 서울시청 설정
2. 앱에서 "산책 시작" 버튼 클릭
3. 위치가 고정되어 있는지 확인

### 시나리오 2: 이동 경로 추적
1. `Freeway Drive` 또는 GPX 파일 사용
2. 앱에서 "산책 시작" 버튼 클릭
3. 카메라가 위치를 따라가는지 확인
4. 경로가 그려지는지 확인

### 시나리오 3: 경로 저장 및 조회
1. 산책 시작 → 이동 → 산책 종료
2. 백엔드에서 경로가 저장되었는지 확인
3. 저장된 경로가 지도에 표시되는지 확인
