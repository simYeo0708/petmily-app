# 🚀 개발 서버 실행 가이드

## "No script URL provided" 에러 해결

시뮬레이터에서 이 에러가 발생하면 Metro 번들러/Expo 개발 서버가 실행되지 않은 것입니다.

---

## ✅ 해결 방법

### 방법 1: Expo 개발 서버 시작 (권장)

**터미널에서 실행:**

```bash
cd Front
npm run dev
# 또는
npm start
```

**그 다음:**
1. Expo 개발 서버가 시작되면 QR 코드와 메뉴가 표시됩니다
2. 시뮬레이터에서 이미 앱이 실행 중이면, 앱을 재시작하세요
3. 또는 Expo 메뉴에서 `i` 키를 눌러 iOS 시뮬레이터를 다시 시작하세요

---

### 방법 2: Xcode에서 직접 실행 (번들러 포함)

**터미널에서 실행:**

```bash
cd Front
npx expo run:ios
```

이 명령어는:
- Metro 번들러를 자동으로 시작합니다
- 시뮬레이터에서 앱을 자동으로 실행합니다
- 개발 모드로 실행되어 실시간 업데이트가 가능합니다

---

### 방법 3: Xcode에서 빌드했지만 서버가 없는 경우

**문제:**
- Xcode에서 직접 빌드했는데 (`expo run:ios --no-bundler`)
- Metro 번들러가 실행되지 않은 경우

**해결:**

1. **터미널 1: Expo 개발 서버 시작**
```bash
cd Front
npm start
```

2. **터미널 2: 시뮬레이터에서 앱 재시작**
   - 시뮬레이터에서 앱을 완전히 종료 (스와이프로 앱 종료)
   - Xcode에서 다시 실행 (Cmd + R)
   - 또는 Expo 메뉴에서 `r` 키를 눌러 앱 재로드

---

## 🔍 확인 방법

### Metro 번들러가 실행 중인지 확인:

```bash
# 포트 8081 (Metro 번들러)
lsof -ti:8081

# 포트 19000 (Expo 서버)
lsof -ti:19000
```

결과가 나오면 실행 중입니다.

---

## ⚠️ 주의사항

1. **Xcode에서 직접 빌드할 때:**
   - `expo run:ios` 명령을 사용하면 자동으로 번들러가 시작됩니다
   - `--no-bundler` 옵션을 사용하면 번들러 없이 빌드되므로, 별도로 서버를 시작해야 합니다

2. **개발 모드 vs 프로덕션 모드:**
   - 개발 모드: Metro 번들러 필요 (실시간 업데이트)
   - 프로덕션 모드: JavaScript 번들이 앱에 포함됨 (서버 불필요)

---

## 📝 빠른 해결 순서

1. **터미널에서:**
```bash
cd Front
npm start
```

2. **Expo 메뉴가 나타나면:**
   - `i` 키: iOS 시뮬레이터 재시작
   - `r` 키: 앱 재로드

3. **시뮬레이터에서:**
   - 앱이 자동으로 연결되어 JavaScript 번들을 로드합니다

---

## 🎯 정상 작동 확인

다음 메시지가 터미널에 표시되면 정상입니다:
```
Metro waiting on exp://192.168.x.x:8081
```

시뮬레이터에서도 앱이 정상적으로 로드되어야 합니다.
